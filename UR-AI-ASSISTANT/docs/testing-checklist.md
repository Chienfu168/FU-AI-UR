# UR AI Assistant 測試清單

本文件用於每次安裝、更新、模組拆分或正式站上傳前的測試檢查。

建議每次修改外掛後，至少依照本清單完成基本測試，再上傳正式網站。

---

## 一、測試原則

UR AI Assistant 屬於公開網站前台互動外掛，測試時應優先確認：

```text
網站不白畫面
後台可正常開啟
前台 shortcode 可正常顯示
FAQ 命中正常
AI 回答正常
問答紀錄正常
資料庫不遺失
錯誤不暴露敏感資訊
二、更新前檢查
1. 備份檔案

更新前請先備份目前穩定版外掛資料夾：

wp-content/plugins/ur-ai-assistant/

建議備份命名：

ur-ai-assistant-backup-v版本號-日期

例如：

ur-ai-assistant-backup-v1.0.0-20260516
2. 備份資料庫

至少備份以下資料表：

wp_ur_ai_faqs
wp_ur_ai_logs
wp_ur_ai_related_pages
wp_ur_ai_popular_questions

如果網站資料表前綴不是 wp_，請依實際前綴調整。

3. 確認目前穩定版

更新前請記錄：

目前外掛版本
目前可正常使用的檔案清單
本次準備修改的檔案
是否有新增資料表或欄位

檢查項目：

沒有 PHP Fatal Error
2. 資料表是否建立

啟用後確認資料表存在：

ur_ai_logs

ur_ai_popular_questions

檢查方式可使用：

Adminer

WP-CLI
3. 資料庫版本是否寫入

確認 option 是否存在：

ur_ai_assistant_db_version

十、回復機制

如果更新後發生錯誤：

1. 立即回復上一版

透過 FTP 或主機檔案管理器，上傳上一個穩定版外掛。

2. 檢查錯誤記錄

開啟或查看：

wp-content/debug.log

檢查是否有：

PHP Fatal error
Class not found
Call to undefined method
Parse error
Cannot redeclare class
3. 不在正式站直接測試大改版

若是模組拆分、資料表改版或 AJAX 流程調整，應先在測試站確認。

十一、最低通過標準

每次正式上線前，至少要通過：

後台不白畫面
前台不白畫面
FAQ 可命中
AI 可回答
Logs 可寫入
回饋可送出
CSV 可匯出
手機版可使用

若其中任何一項失敗，不建議上傳正式站。

## 這個檔案的設計重點

### 1. 這是正式站更新前的安全清單

未來每次要上傳正式網站前，都可以依這份逐項檢查。

### 2. 特別針對曾經遇過的白畫面風險

清單中已包含：

```text
Class not found
Parse error
View 缺漏
AJAX 錯誤
資料表缺漏

這些都是 WordPress 外掛模組化最容易出事的地方。

3. 測試完再進正式站

這份文件的核心精神是：先測試站，後正式站；先小改，後大改。
